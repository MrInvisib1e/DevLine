How should the graph nodes relate to each other?

Pick the relationship model that feels right for DevFlow's memory

A

Typed Edges on Each Node

Every node declares its relationships explicitly by type. Simple, predictable, easy for df-explain to traverse.

{
  "name": "Comment",
  "type": "entity",
  "file": "Entities/Comment.cs",
  "depends_on": ["User", "Story"],
  "used_by": ["CommentService"],
  "persisted_in": "comments",
  "contracts": ["CommentCreatedEvent"]
}
B

Universal Edge List

All relationships live in a single edges[] array. Flexible, queryable, but verbose. Good if edge types will grow over time.

"edges": [
  { "from": "Comment", "to": "User", "rel": "depends_on" },
  { "from": "Comment", "to": "Story", "rel": "depends_on" },
  { "from": "CommentService", "to": "Comment", "rel": "uses" },
  { "from": "Comment", "to": "comments", "rel": "persisted_in" }
]
C

Hybrid — Typed Edges + Intent

Typed edges like A, but each relationship also carries an intent — the why behind the connection. Richest context for AI reasoning.

{
  "name": "Comment",
  "depends_on": [
    { "node": "User", "intent": "author ownership" },
    { "node": "Story", "intent": "parent context" }
  ],
  "intent": "Soft-deletable — hide, never purge"
}
