# DevFlow v4 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Redesign DevFlow into a clean, generic, single-library development workflow tool powered by codebase-memory-mcp — eliminating custom graph infrastructure, consolidating all skills into one library, adding behavior contracts, quality hooks, research tiers, benchmarking, multi-project orchestration, and removing all project-specific hardcoding.

**Architecture:** DevFlow v4 replaces ~2400 lines of custom bash graph infrastructure with a thin wrapper around the `codebase-memory-mcp` MCP server (14 tools, 155 languages, Cypher queries, semantic search). All 29 skills live in one library (no external Superpowers dependency). Memory is simplified to a single `.devflow/memory.md` per project. The full slash command namespace uses `df-` prefix.

**Tech Stack:** Bash (bin scripts), Markdown (skills), SQLite via codebase-memory-mcp, Node.js (df-benchmark), codebase-memory-mcp (C static binary, MIT)

### Task 1.4: Create `df-check` quality hooks script

**Files:**
- Create: `bin/df-check`

- [ ] **Step 1: Create bin/df-check**

```bash
cat > bin/df-check << 'SCRIPT'
#!/usr/bin/env bash
# df-check — run quality hooks (lint/format/typecheck) for current project
# Usage: df-check [--lint-only] [--format-only] [--typecheck-only] [--no-fix]
# Exit codes: 0=all pass, 1=typecheck failed (blocking), 2=lint/format issues (advisory)

set -euo pipefail

DEVFLOW_DIR="$(git rev-parse --show-toplevel 2>/dev/null)/.devflow"
CONFIG="$DEVFLOW_DIR/config.json"
LINT_ONLY=false
FORMAT_ONLY=false
TYPECHECK_ONLY=false
NO_FIX=false
EXIT_CODE=0

for arg in "$@"; do
  case "$arg" in
    --lint-only) LINT_ONLY=true ;;
    --format-only) FORMAT_ONLY=true ;;
    --typecheck-only) TYPECHECK_ONLY=true ;;
    --no-fix) NO_FIX=true ;;
  esac
done

# Read quality_hooks from config if available
if [ -f "$CONFIG" ]; then
  LINT_CMD=$(python3 -c "import json,sys; c=json.load(open('$CONFIG')); h=c.get('quality_hooks',{}); print(h.get('lint',''))" 2>/dev/null || echo "")
  FORMAT_CMD=$(python3 -c "import json,sys; c=json.load(open('$CONFIG')); h=c.get('quality_hooks',{}); print(h.get('format',''))" 2>/dev/null || echo "")
  TYPECHECK_CMD=$(python3 -c "import json,sys; c=json.load(open('$CONFIG')); h=c.get('quality_hooks',{}); print(h.get('typecheck',''))" 2>/dev/null || echo "")
else
  # Auto-detect from project files
  ROOT="$(git rev-parse --show-toplevel 2>/dev/null)"
  if [ -f "$ROOT/package.json" ]; then
    LINT_CMD="npx eslint --fix ."
    FORMAT_CMD="npx prettier --write ."
    if [ -f "$ROOT/tsconfig.json" ]; then
      TYPECHECK_CMD="npx tsc --noEmit"
    fi
  elif ls "$ROOT"/*.csproj "$ROOT"/**/*.csproj 2>/dev/null | head -1 | grep -q csproj; then
    LINT_CMD=""
    FORMAT_CMD="dotnet format"
    TYPECHECK_CMD="dotnet build --no-restore -warnaserror"
  elif [ -f "$ROOT/Cargo.toml" ]; then
    LINT_CMD="cargo clippy -- -D warnings"
    FORMAT_CMD="cargo fmt"
    TYPECHECK_CMD="cargo check"
  elif [ -f "$ROOT/go.mod" ]; then
    LINT_CMD="golangci-lint run"
    FORMAT_CMD="gofmt -w ."
    TYPECHECK_CMD="go build ./..."
  fi
fi

# Run typecheck (blocking — exit 1 on failure)
if [ "$LINT_ONLY" = false ] && [ "$FORMAT_ONLY" = false ] && [ -n "${TYPECHECK_CMD:-}" ]; then
  echo "[df-check] typecheck: $TYPECHECK_CMD"
  if ! eval "$TYPECHECK_CMD"; then
    echo "[df-check] ❌ TYPECHECK FAILED (blocking)"
    exit 1
  fi
  echo "[df-check] ✅ typecheck passed"
fi

# Run format (advisory — auto-fix if possible)
if [ "$LINT_ONLY" = false ] && [ "$TYPECHECK_ONLY" = false ] && [ -n "${FORMAT_CMD:-}" ]; then
  echo "[df-check] format: $FORMAT_CMD"
  if [ "$NO_FIX" = true ]; then
    FORMAT_CMD="${FORMAT_CMD/--write/--check}"
    FORMAT_CMD="${FORMAT_CMD/ -w / --dry-run }"
  fi
  if ! eval "$FORMAT_CMD" 2>/dev/null; then
    echo "[df-check] ⚠️  format issues found (advisory)"
    EXIT_CODE=2
  fi
fi

# Run lint (advisory — auto-fix if possible)
if [ "$FORMAT_ONLY" = false ] && [ "$TYPECHECK_ONLY" = false ] && [ -n "${LINT_CMD:-}" ]; then
  echo "[df-check] lint: $LINT_CMD"
  if [ "$NO_FIX" = true ]; then
    LINT_CMD="${LINT_CMD/ --fix/}"
  fi
  if ! eval "$LINT_CMD" 2>/dev/null; then
    echo "[df-check] ⚠️  lint issues found (advisory)"
    EXIT_CODE=2
  fi
fi

exit $EXIT_CODE
SCRIPT
chmod +x bin/df-check
```

- [ ] **Step 2: Write the new df-init**

```bash
cat > bin/df-init << 'SCRIPT'
#!/usr/bin/env bash
# df-init — Initialize DevFlow for current project
# Usage: df-init [--scan] [--write-memory] [--reset] [--orchestrator]
# Requires: codebase-memory-mcp running as MCP server

set -euo pipefail

MODE="init"
for arg in "$@"; do
  case "$arg" in
    --scan) MODE="scan" ;;
    --write-memory) MODE="write-memory" ;;
    --reset) MODE="reset" ;;
    --orchestrator) MODE="orchestrator" ;;
  esac
done

# --- Project root detection ---
# 1. .devflow/ in current directory? → use it
# 2. .devflow/ at git root? → use it
# 3. Neither? → create .devflow/ in current directory
# NEVER go above git root.

GIT_ROOT="$(git rev-parse --show-toplevel 2>/dev/null)" || {
  echo "Error: not inside a git repository" >&2; exit 1
}

if [ -d ".devflow" ]; then
  PROJECT_ROOT="$(pwd)"
elif [ -d "$GIT_ROOT/.devflow" ]; then
  PROJECT_ROOT="$GIT_ROOT"
else
  PROJECT_ROOT="$(pwd)"
fi

DEVFLOW_DIR="$PROJECT_ROOT/.devflow"
CONFIG="$DEVFLOW_DIR/config.json"
MEMORY="$DEVFLOW_DIR/memory.md"

# --- Stack detection ---
detect_stack() {
  local root="$1"
  local runtime="" frontend=""
  for f in "$root/package.json" "$root/vite.config.ts" "$root/vite.config.js" \
            "$root/next.config.js" "$root/next.config.ts" \
            $(find "$root" -maxdepth 3 -name "*.csproj" -o -name "*.sln" 2>/dev/null | head -1) \
            "$root/requirements.txt" "$root/go.mod" "$root/Cargo.toml" \
            "$root/Gemfile" "$root/pyproject.toml"; do
    [ -f "$f" ] || continue
    case "$(basename "$f")" in
      package.json) runtime="nodejs" ;;
      vite.config.*) frontend="sveltekit" ;;
      next.config.*) frontend="nextjs" ;;
      *.csproj|*.sln) runtime="dotnet" ;;
      requirements.txt|pyproject.toml) runtime="python" ;;
      go.mod) runtime="go" ;;
      Cargo.toml) runtime="rust" ;;
      Gemfile) runtime="ruby" ;;
    esac
  done
  echo "{\"runtime\":\"${runtime:-unknown}\",\"frontend\":\"${frontend:-null}\"}"
}

# --- Check codebase-memory-mcp is available ---
check_mcp() {
  if ! command -v codebase-memory-mcp &>/dev/null; then
    echo "Error: codebase-memory-mcp not found." >&2
    echo "Install with: df-install --mcp" >&2
    exit 1
  fi
}

case "$MODE" in
  scan)
    check_mcp
    STACK=$(detect_stack "$PROJECT_ROOT")
    # Return JSON for skill consumption
    echo "{\"project_root\":\"$PROJECT_ROOT\",\"git_root\":\"$GIT_ROOT\",\"stack\":$STACK}"
    ;;

  write-memory|init)
    check_mcp
    mkdir -p "$DEVFLOW_DIR"

    # Index repository via codebase-memory-mcp CLI
    echo "[df-init] Indexing repository..."
    codebase-memory-mcp cli index_repository "{\"repo_path\":\"$PROJECT_ROOT\"}" || {
      echo "Error: codebase-memory-mcp indexing failed" >&2; exit 1
    }

    # Get architecture overview
    echo "[df-init] Generating memory..."
    ARCH=$(codebase-memory-mcp cli get_architecture "{}" 2>/dev/null || echo "{}")

    # Get top nodes by graph degree (proxy for PageRank)
    TOP_NODES=$(codebase-memory-mcp cli search_graph \
      "{\"limit\":50,\"min_degree\":2}" 2>/dev/null || echo "[]")

    # Detect stack
    STACK=$(detect_stack "$PROJECT_ROOT")

    # Write config.json if not present
    if [ ! -f "$CONFIG" ]; then
      SERVICE=$(basename "$PROJECT_ROOT")
      cat > "$CONFIG" << JSON
{
  "service": "$SERVICE",
  "mode": "project",
  "stack": $STACK,
  "last_synced": "$(git rev-parse HEAD 2>/dev/null || echo 'unknown')",
  "classifiers": [],
  "review_checks": ["naming","test-coverage","unclassified","impact-radius","dead-code","clone-detection"],
  "auto_skills": [],
  "quality_hooks": {}
}
JSON
    else
      # Update last_synced
      TMP=$(mktemp)
      python3 -c "
import json, sys
c = json.load(open('$CONFIG'))
c['last_synced'] = '$(git rev-parse HEAD 2>/dev/null || echo unknown)'
json.dump(c, open('$TMP','w'), indent=2)
"
      mv "$TMP" "$CONFIG"
    fi

    # Render memory.md
    python3 - "$MEMORY" "$ARCH" "$TOP_NODES" << 'PYEOF'
import json, sys, textwrap

memory_path = sys.argv[1]
arch = json.loads(sys.argv[2]) if sys.argv[2] != '{}' else {}
nodes = json.loads(sys.argv[3]) if sys.argv[3] != '[]' else []

lines = ["# DevFlow Memory\n"]

# Architecture section
lines.append("## Architecture\n")
if arch:
  if arch.get("languages"):
    lines.append(f"**Languages:** {', '.join(arch['languages'])}\n")
  if arch.get("entry_points"):
    lines.append(f"**Entry Points:** {', '.join(arch['entry_points'][:5])}\n")
  if arch.get("clusters"):
    lines.append(f"**Modules:** {', '.join(arch['clusters'][:5])}\n")
  if arch.get("hotspots"):
    lines.append(f"**Hotspots:** {', '.join(str(h) for h in arch['hotspots'][:5])}\n")

lines.append("\n## Top Nodes\n")
for n in nodes[:50]:
  name = n.get("name", "?")
  label = n.get("label", "?")
  file_ = n.get("file", "")
  lines.append(f"- **{name}** [{label}] — {file_}\n")

content = "\n".join(lines)
# Cap at ~2500 tokens (~10000 chars)
if len(content) > 10000:
  content = content[:9900] + "\n\n[truncated for token budget]\n"

with open(memory_path, "w") as f:
  f.write(content)
print(f"[df-init] memory.md written ({len(content)} chars)")
PYEOF

    echo "[df-init] ✅ DevFlow initialized at $DEVFLOW_DIR"
    ;;

  reset)
    rm -rf "$DEVFLOW_DIR"
    echo "[df-init] Reset complete. Run /df-init to reinitialize."
    ;;

  orchestrator)
    mkdir -p "$DEVFLOW_DIR"
    # Discover child .devflow/ directories
    PROJECTS="[]"
    while IFS= read -r -d '' dir; do
      path="${dir%/.devflow}"
      name="$(basename "$path")"
      rel="$(realpath --relative-to="$PROJECT_ROOT" "$path")"
      PROJECTS=$(echo "$PROJECTS" | python3 -c "
import json,sys
p=json.load(sys.stdin)
p.append({'name':'$name','path':'$rel'})
print(json.dumps(p))
")
    done < <(find "$PROJECT_ROOT" -mindepth 2 -maxdepth 4 -name ".devflow" -type d -print0)

    cat > "$CONFIG" << JSON
{
  "service": "$(basename "$PROJECT_ROOT")",
  "mode": "orchestrator",
  "projects": $PROJECTS,
  "last_synced": "$(git rev-parse HEAD 2>/dev/null || echo 'unknown')"
}
JSON
    echo "[df-init] ✅ Orchestrator initialized with projects: $PROJECTS"
    ;;
esac
SCRIPT
chmod +x bin/df-init
```

### Task 2.2: Rewrite `df-explain` as thin MCP wrapper

**Files:**
- Modify: `bin/df-explain`

- [ ] **Step 1: Write the new df-explain**

```bash
cat > bin/df-explain << 'SCRIPT'
#!/usr/bin/env bash
# df-explain — Query the codebase knowledge graph via codebase-memory-mcp
# Usage:
#   df-explain <query>              — search for a symbol/concept
#   df-explain --rank [--budget N]  — top N nodes by degree
#   df-explain --diff HEAD~5 HEAD   — what changed between commits
#   df-explain --node <file>        — node for a file + 1 level edges
#   df-explain --impact             — detect_changes blast radius
#   df-explain --dead-code          — find zero-caller functions
#   df-explain --clones             — find near-clone pairs
# Options:
#   --project <name>                — scope to a specific project (orchestrator mode)

set -euo pipefail

PROJECT=""
BUDGET=512
ARGS=()

while [[ $# -gt 0 ]]; do
  case "$1" in
    --project) PROJECT="$2"; shift 2 ;;
    --budget) BUDGET="$2"; shift 2 ;;
    *) ARGS+=("$1"); shift ;;
  esac
done

PROJECT_ARG=""
if [ -n "$PROJECT" ]; then
  PROJECT_ARG="\"project\":\"$PROJECT\","
fi

check_mcp() {
  command -v codebase-memory-mcp &>/dev/null || {
    echo "Error: codebase-memory-mcp not found. Install with: df-install --mcp" >&2
    exit 1
  }
}

case "${ARGS[0]:-search}" in
  --rank)
    check_mcp
    codebase-memory-mcp cli search_graph \
      "{${PROJECT_ARG}\"limit\":$BUDGET,\"min_degree\":1}" \
      | python3 -c "
import json,sys
nodes=json.load(sys.stdin)
for n in nodes:
  print(f\"{n.get('name','?')} [{n.get('label','?')}] — {n.get('file','')} (degree: {n.get('degree',0)})\")
"
    ;;

  --diff)
    check_mcp
    FROM="${ARGS[1]:-HEAD~1}"
    TO="${ARGS[2]:-HEAD}"
    codebase-memory-mcp cli query_graph \
      "{${PROJECT_ARG}\"query\":\"MATCH (n) WHERE n.file_changed_in >= '$FROM' RETURN n LIMIT 50\"}" \
      2>/dev/null || echo "(diff query not supported in this version)"
    ;;

  --node)
    check_mcp
    FILE="${ARGS[1]}"
    codebase-memory-mcp cli search_graph \
      "{${PROJECT_ARG}\"file_pattern\":\"$FILE\",\"limit\":20}" | python3 -c "
import json,sys
nodes=json.load(sys.stdin)
for n in nodes:
  print(f\"{n.get('name','?')} [{n.get('label','?')}] — {n.get('file','')}\")"
    echo "--- Outbound ---"
    codebase-memory-mcp cli trace_call_path \
      "{${PROJECT_ARG}\"function_name\":\"$(basename "${FILE%.*}")\",\"direction\":\"outbound\",\"depth\":1}" \
      2>/dev/null || echo "(trace not available)"
    echo "--- Inbound ---"
    codebase-memory-mcp cli trace_call_path \
      "{${PROJECT_ARG}\"function_name\":\"$(basename "${FILE%.*}")\",\"direction\":\"inbound\",\"depth\":1}" \
      2>/dev/null || echo "(trace not available)"
    ;;

  --impact)
    check_mcp
    codebase-memory-mcp cli detect_changes "{${PROJECT_ARG:-}}" | python3 -c "
import json,sys
data=json.load(sys.stdin)
symbols=data.get('affected_symbols',[])
print(f'Affected symbols: {len(symbols)}')
for s in symbols:
  risk=s.get('risk','unknown')
  print(f'  [{risk.upper()}] {s.get(\"name\",\"?\")} — {s.get(\"file\",\"\")}')"
    ;;

  --dead-code)
    check_mcp
    codebase-memory-mcp cli query_graph \
      "{${PROJECT_ARG}\"query\":\"MATCH (f:Function) WHERE NOT ()-[:CALLS]->(f) AND NOT f.name STARTS WITH 'test' RETURN f.name, f.file LIMIT 50\"}" | python3 -c "
import json,sys
data=json.load(sys.stdin)
rows=data.get('rows',[])
print(f'Dead code candidates: {len(rows)}')
for r in rows:
  print(f'  {r.get(\"f.name\",\"?\")} — {r.get(\"f.file\",\"\")}')"
    ;;

  --clones)
    check_mcp
    codebase-memory-mcp cli query_graph \
      "{${PROJECT_ARG}\"query\":\"MATCH (a)-[r:SIMILAR_TO]->(b) WHERE r.score > 0.8 RETURN a.name, b.name, r.score ORDER BY r.score DESC LIMIT 20\"}" | python3 -c "
import json,sys
data=json.load(sys.stdin)
rows=data.get('rows',[])
print(f'Near-clone pairs (score > 0.8): {len(rows)}')
for r in rows:
  print(f'  {r.get(\"a.name\",\"?\")} ≈ {r.get(\"b.name\",\"?\")} (score: {r.get(\"r.score\",0):.2f})')"
    ;;

  *)
    # Default: semantic + structural search for query term
    check_mcp
    QUERY="${ARGS[0]}"
    echo "=== Structural search: $QUERY ==="
    codebase-memory-mcp cli search_graph \
      "{${PROJECT_ARG}\"name_pattern\":\"$QUERY\",\"limit\":20}" | python3 -c "
import json,sys
nodes=json.load(sys.stdin)
for n in nodes:
  print(f\"  {n.get('name','?')} [{n.get('label','?')}] — {n.get('file','')}\")"
    echo ""
    echo "=== Call paths: $QUERY ==="
    codebase-memory-mcp cli trace_call_path \
      "{${PROJECT_ARG}\"function_name\":\"$QUERY\",\"direction\":\"both\",\"depth\":2}" \

# /df-benchmark — A/B Test Skills

Compare two versions of a skill (or rule) against falsifiable assertions using isolated worktrees.

## When to Use
- Before changing an existing skill (validate improvement)
- After absorbing a new skill (verify it's better than baseline)
- When debugging why a skill produces inconsistent results

## Required
- `df-benchmark` script: `df-benchmark --skill <name> --scenario <file> [--variant <modified-skill-path>]`

## Workflow

### Step 1 — Define scenario (T3 Gate)

A scenario is a JSON file at `.devflow/benchmarks/<name>.json`:
```json
{
  "name": "fix-empty-body-validation",
  "description": "Agent should fix a 500 error on empty request body",
  "git_state": "HEAD~3",
  "task": "Fix: POST /api/comments returns 500 when body is empty",
  "assertions": [
    { "type": "file_modified", "path": "src/CommentsService.cs" },
    { "type": "test_passes", "command": "dotnet test --filter EmptyBody" },
    { "type": "cycles_le", "value": 2, "description": "Fixed in ≤2 cycles" },
    { "type": "file_not_modified", "path": "src/AuthService.cs", "description": "Scope fence respected" }
  ]
}
```

Present scenario to user. T3 Gate: confirm assertions are falsifiable before running.

### Step 2 — Run baseline (T1 Silent)

1. Create worktree A at scenario `git_state`
2. Apply current skill version
3. Run task with agent in worktree A
4. Capture: files modified, tests run, outcome, cycle count

### Step 3 — Run variant (T1 Silent)

1. Create worktree B at same `git_state`
2. Apply variant skill (from `--variant` path, or modified version)
3. Run same task with agent in worktree B
4. Capture same metrics

### Step 4 — Grade and verdict (T2 Inform)

For each assertion: PASS / FAIL in A vs B.

```
BENCHMARK RESULTS: fix skill — empty-body scenario
─────────────────────────────────────────────────
                  BASELINE    VARIANT
file_modified     ✅ PASS     ✅ PASS
test_passes       ✅ PASS     ✅ PASS
cycles_le_2       ❌ FAIL(3)  ✅ PASS(2)
scope_fence       ✅ PASS     ✅ PASS

VERDICT: VARIANT WINS — 1 improvement, 0 regressions
Recommend: promote variant to production
```

### Step 5 — Cleanup (T1 Silent)

Remove both worktrees. Archive results to `.devflow/benchmarks/<name>-result-<timestamp>.json`.

## Scenario Assertion Types

| Type | Description |
|------|-------------|
| `file_modified` | File must appear in diff |
| `file_not_modified` | File must NOT appear in diff (scope fence) |
| `test_passes` | Command must exit 0 |
| `test_fails` | Command must exit non-0 |
| `cycles_le` | Cycle count ≤ value |
| `output_contains` | Agent output must contain string |
| `output_not_contains` | Agent output must NOT contain string |
SKILL
```
